# deploy 目录 · 批处理脚本说明

本目录下的 `.bat` 文件均为 **Windows 双击入口**，内部会切换到仓库根目录并调用同名 PowerShell 脚本（`.ps1`）。失败时窗口会暂停，便于查看报错。

**环境要求**

- Windows 10/11，已安装 **PowerShell 5.1+**（系统自带）
- 首次使用前需按各脚本说明准备对应的 `.env` 配置文件（见 `*.example` 模板）
- 敏感配置（`sync.env`、`quiz-live-relay.env` 等）已被 `.gitignore` 忽略，勿提交 Git

---

## 脚本一览

| 批处理 | 对应 PowerShell | 作用 |
|--------|-----------------|------|
| `sync.bat` | `sync.ps1` | 将仓库静态文件同步到腾讯云 COS |
| `quiz-live-relay.bat` | `quiz-live-relay.ps1` | 通过 SSH 更新远端 quiz-live WebSocket 中继 |
| `quiz-live-deploy.bat` | `quiz-live-deploy.ps1` | **一键完整部署**：先 COS 静态站，再 relay |

## sync.bat — 静态站上传到 COS

### 用途

把 Monorepo **仓库根目录**下的站点文件（HTML、CSS、JS、assets 等）增量同步到腾讯云对象存储（COS）。本地 `index.html` 会映射到桶内 `PREFIX/index.html`（例如 `/sfkdoc/index.html`），不会多出一层 `Project/` 目录。

同步前会自动：

1. 读取 `deploy/sync.env` 并生成 `deploy/cos.yaml`（供 coscli 使用）
2. 若本机没有 coscli，尝试下载到 `deploy/tools/coscli.exe`
3. 将 `contents/data/*.json` 复制到根目录 `data/` 再上传

**不会上传**的内容包括：`deploy/`、`.git/`、`contents/`、`docs/`、`tools/`、`.codex/`、`.cursor/`、根目录 Agent 配置与 `*.md` 等（见 `sync.ps1` 内 `--exclude` 列表）。

### 配置文件

复制模板并填写密钥：

```text
deploy/sync.env.example  →  deploy/sync.env
```

必填项：`SECRET_ID`、`SECRET_KEY`、`BUCKET`、`REGION`  
可选项：`PREFIX`、`PUBLIC_BASE_URL`、`COSCLI_PATH`

403 权限错误时，参考 `deploy/cam-policy.sfkdoc.example.json` 为 CAM 子账号配置桶读写策略。

### 用法

在资源管理器中双击 `sync.bat`，或在仓库根目录执行：

```bat
deploy\sync.bat
```

**可选参数**（传给 `sync.ps1`）：

| 参数 | 说明 |
|------|------|
| `-DryRun` | 仅打印将要执行的 coscli 命令，实际上传 |
| `-DeleteRemote` | 同步时删除 COS 上本地已不存在的远端文件，并**额外清理**旧目录结构遗留（见下） |

**关于 `-DeleteRemote` 与旧文件：**

coscli 的 `--exclude`（如 `*.md`、`tools/**`、`docs/**`、`.codex/**`）会同时阻止上传**和**镜像删除，因此根目录的 `BILIBILI_EMBED_GUIDE.md`、`AGENTS.md`、`.cursorrules`、`tools/` 等即使用 `-DeleteRemote` 也可能删不掉。
脚本在 `-DeleteRemote` 时会：

1. 清空本地 coscli snapshot 缓存（避免 `destination num: 0` 导致跳过对比）
2. 执行 `coscli sync --delete`
3. **额外 `coscli rm`** 已知旧路径：`guides/`、根目录 `*.md` 指南、`tools/`、`outline.md`、`share-pages.json` 等

请确认控制台当前浏览的是 **`PREFIX` 对应目录**（如 `sfkdoc/`），不是桶根目录或其它前缀下的历史文件。

示例：

```bat
deploy\sync.bat -DryRun
deploy\sync.bat -DeleteRemote
```

### 成功后

终端会打印对外访问地址，例如：

- 主 deck：`PUBLIC_BASE_URL/index.html`
- 子站：`PUBLIC_BASE_URL/summerschool/` 等

## quiz-live-relay.bat — 部署 WebSocket 中继

### 用途

通过 **SSH + SCP** 将 quiz-live 实时答题所需文件上传到远端服务器（默认东京 Lighthouse `/opt/quiz-live`），并用 **pm2** 重启 relay 进程。

上传文件：

- `quiz-live/scripts/quiz-ws-relay.js`
- `quiz-live/scripts/relay/*.js`
- `quiz-live/data/quiz/*.json`（含 `question_cfg.json`、各 `questions_*.json`）
- `quiz-live/data/levels.json`
- `quiz-live/data/broadcast-config.json`

**不会覆盖**远端运行时生成的 `room-state.json`（房间状态）。

远端 relay 默认监听 **8082** 端口（WebSocket + HTTP 健康检查）。

### 前置条件

1. 服务器已执行过一次 `deploy/quiz-live-server-setup.sh`（安装 Node.js、pm2 并启动 relay）
2. 腾讯云 Lighthouse 防火墙已放行 **TCP 8082**
3. 本机已安装 **OpenSSH 客户端**（Windows「可选功能」中启用，或使用 Git 自带的 ssh/scp）

### 配置文件

```text
deploy/quiz-live-relay.env.example  →  deploy/quiz-live-relay.env
```

必填项：`RELAY_HOST`（公网 IP 或域名）、`RELAY_USER`（如 `ubuntu`）  
常用可选项：`RELAY_SSH_KEY`（私钥路径）、`RELAY_SSH_PORT`、`RELAY_APP_DIR`、`RELAY_PM2_NAME`

### 用法

```bat
deploy\quiz-live-relay.bat
```

**可选参数**：

| 参数 | 说明 |
|------|------|
| `-DryRun` | 列出将要上传的文件与 pm2 重启命令，不实际连接服务器 |
| `-NoRestart` | 上传文件但不执行 `pm2 restart`（适合仅更新题库 JSON） |

示例：

```bat
deploy\quiz-live-relay.bat -DryRun
deploy\quiz-live-relay.bat -NoRestart
```

## quiz-live-deploy.bat — quiz-live 完整部署

### 用途

按顺序执行 **两步部署**，适合发布 quiz-live 相关改动时一次跑完：

1. **Step 1/2**：调用 `sync.ps1` — 更新 COS 上的静态页面（`quiz-live/` 下的 HTML、CSS、JS 等）
2. **Step 2/2**：调用 `quiz-live-relay.ps1` — 更新远端 WebSocket 中继与题库数据

因此需要同时准备好 **`sync.env`** 和 **`quiz-live-relay.env`**。

### 用法

```bat
deploy\quiz-live-deploy.bat
```

**可选参数**（会传递给对应子脚本）：

| 参数 | 说明 |
|------|------|
| `-DryRun` | 两步均只做预览，不上传 |
| `-StaticOnly` | 仅执行 COS 静态同步（跳过 relay） |
| `-RelayOnly` | 仅执行 relay 部署（跳过 COS） |
| `-NoRestart` | relay 步骤上传后不重启 pm2 |

示例：

```bat
deploy\quiz-live-deploy.bat -DryRun
deploy\quiz-live-deploy.bat -StaticOnly
deploy\quiz-live-deploy.bat -RelayOnly -NoRestart
```

| 场景 | 推荐命令 |
|------|----------|
| 只改了幻灯片 / 主站 / 子站 HTML | `deploy\sync.bat` |

## 故障排查

### sync.bat 报「Missing deploy/sync.env」

复制 `sync.env.example` 为 `sync.env` 并填写腾讯云 CAM 密钥与桶信息。

### sync.bat 报 COS 403

检查 `SECRET_ID` / `SECRET_KEY` 是否为具备目标桶读写权限的子账号；对照 `cam-policy.sfkdoc.example.json` 调整 CAM 策略。

### quiz-live-relay.bat 报 ssh / scp not found

在 Windows「设置 → 应用 → 可选功能」中安装 **OpenSSH 客户端**，或确保 Git 的 `usr\bin` 在 PATH 中。

### quiz-live-relay.bat 报 RELAY_SSH_KEY not found

在 `quiz-live-relay.env` 中填写正确的 `.pem` 私钥路径（Lighthouse 控制台下载）。

### 上传成功但观众连不上 WebSocket

1. 确认 Lighthouse 防火墙已放行 **8082**
2. 在服务器上执行 `pm2 logs quiz-relay` 查看 relay 日志
3. 公网测试：`curl http://<RELAY_HOST>:8082/` 应返回 `quiz-live relay OK`

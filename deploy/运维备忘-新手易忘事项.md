# 运维备忘 · 新手易忘事项

面向本仓库 **quiz-live 公网部署**（COS 静态页 + Lighthouse WebSocket）。与 Agent 协作前，可先扫一遍本文；当前机器上的实际配置见 [`服务器状态快照.md`](服务器状态快照.md)。

---

## 1. 两套域名、两张证书，别混

| 域名 | 干什么 | 证书绑在哪 | 证书 ID（2026-06） |
|------|--------|------------|-------------------|
| `sska.site` | 静态 HTML（`/sfkdoc/...`） | **COS** 自定义源站域名 | `YDmj75H4`（含 `www.sska.site`） |
| `ws.sska.site` | WebSocket 中继 | **Lighthouse Nginx** 443 | `YDnZlgBX` |

- 只绑 COS 不会让 `wss://` 生效；只配 Nginx 不会让 `https://sska.site/...` 生效。
- 免费证 **90 天**一轮；控制台可开 **自动续费**（免费、不另收费），但 **`ws.sska.site` 续期后仍要手动下载并更新 Nginx 上的文件**。

---

## 2. HTTP 能开 ≠ HTTPS 已好

- `http://sska.site/...` 不校验证书，所以「页面能打开」不代表 HTTPS 没问题。
- HTTPS 要用 `openssl` 看 **Subject 是否是 `sska.site`**，不能仍是 `*.cos.ap-osaka.myqcloud.com`：

```bash
openssl s_client -connect sska.site:443 -servername sska.site </dev/null 2>/dev/null | openssl x509 -noout -subject
```

- COS 绑证后官方说约 **30 分钟**生效；久仍为默认 COS 证 → 解绑重绑或等更久。

---

## 3. DNS 与 SSL 验证是两回事

- **CNAME 绿勾**（`sska.site` → `cos-website...`）只说明网站解析对了。
- 申请证书时还要单独加 **`_dnsauth` TXT**：
  - 主站：`sska.site` 区 → 主机 `_dnsauth`
  - 子域 WS：`sska.site` 区 → 主机 `_dnsauth.ws`（或子域区 `_dnsauth`）
- `nslookup` 能查到 TXT，腾讯云按钮仍报「未检测到」→ 常见是 **延迟**；等 15～30 分钟，勿狂点「验证」（会限频）。
- DNSPod 里 TXT 记录的 **线路类型请用「默认」**，不要只选境内/境外。

---

## 4. 浏览器怎么连 WebSocket

- 静态页在 **HTTPS** 时，前端会自动用 **`wss://ws.sska.site`**（**不带 `:8082`**）。
- Nginx 在 **443** 终止 TLS，反代到本机 **`127.0.0.1:8082`**（pm2 里的 `quiz-relay`）。
- 调试 WS：F12 → **Network → WS** → **先开 F12 再刷新** → 看状态 **101**。
- 页面上显示 **「已连接」** 就说明 WS 已通，不必强求 Network 里有记录。

## 6. SSH / 服务器上常用命令

```bash
# 看 relay 日志（退出按 Ctrl+C，不是卡死）
pm2 logs quiz-relay

pm2 status
pm2 restart quiz-relay

# relay 健康（本机）
curl http://127.0.0.1:8082/

# WSS + Nginx（公网）
curl https://ws.sska.site/

# 检查 Nginx 配置
sudo nginx -t
sudo systemctl reload nginx
```

- `pm2 logs` 后没有 `ubuntu@...:` 提示符是正常的，**Ctrl+C** 回到 shell。

---

## 7. 证书文件在服务器上的位置（ws）

仅 **`ws.sska.site`** 需要你把 zip 解压后拷到机器上：

```text
/etc/nginx/ssl/ws.sska.site.pem    ← bundle.crt 复制过来
/etc/nginx/ssl/ws.sska.site.key    ← .key 复制过来
```

Nginx 站点配置：`/etc/nginx/sites-available/ws.sska.site`

续期后：**重新下载 → 覆盖上述两个文件 → `sudo systemctl reload nginx`**。

## 8. 防火墙端口

| 端口 | 用途 | 建议 |
|------|------|------|
| 443 | `wss://ws.sska.site` | 必须放行 |
| 80 | HTTP→HTTPS 跳转 | 建议放行 |
| 8082 | relay 直连 | WSS 配好后可对公网关闭，仅本机 Nginx 访问 |
| 22 | SSH | 部署脚本需要 |

## 9. 容易踩的小坑

- **`/favicon.ico` 403**：浏览器默认请求域名根路径；quiz-live 页应在 HTML 里写 `<link rel="icon" href="../assets/favicon.svg">`，不能指望根路径有 `favicon.ico`。
- **Nginx 警告 `conflicting server name "ws.sska.site"`**：同一域名被配置了两次，用 `grep -r ws.sska.site /etc/nginx/sites-enabled/` 清理重复项。
- **静态网站「强制 HTTPS」**：COS 证书未生效前可先关掉，避免异常跳转。
- **默认房间码**：`quiz-live/data/default-room.txt` 现为 `SFKMAIN`；链接要带 `?room=SFKMAIN`（三端一致）。

# 从 Project 新建演示 · 说明

脚本位于 **`Project/新开项目工具/`**，从现行参考工程 [`Project/`](../) 一键复制出**新的独立 deck 目录**（创建在工坊根目录，与 `Project/` 同级）。

| 脚本 | 说明 |
|------|------|
| [`new-deck-from-template.bat`](new-deck-from-template.bat) | **推荐** — 主入口 |
| [`new-deck-from-template我的新演示.bat`](new-deck-from-template我的新演示.bat) | 兼容旧文件名，转发到主脚本 |
| [`scaffold-cleanup.ps1`](scaffold-cleanup.ps1) | 复制后清理课例残留（由 bat 调用，一般无需手动运行） |

---

## 用法

在 **`Project/新开项目工具/`** 目录双击 `new-deck-from-template.bat`，或在终端运行：

```bat
cd Project\新开项目工具
new-deck-from-template.bat 你的演示文件夹名
```

示例：

```bat
new-deck-from-template.bat 2026春季招生宣讲
```

不带参数时，脚本会提示输入文件夹名。新目录创建在**工坊根目录**下，例如：

```text
幻灯片生成工坊/2026春季招生宣讲/
```

---

## 脚本做了什么

| 步骤 | 说明 |
|------|------|
| 1. 复制工程骨架 | 从 `Project/` 递归复制设计系统、脚本、文档、部署脚手架等 |
| 2. 排除课例内容 | 不复制 `summerschool/`、`quiz-live/`、`新开项目工具/`、`.git`、任意 `pdf-extracted/`、就业课大图、述职 PDF 等 |
| 3. 写入最小示例页 | 用 `Project/tools/dev/new-deck-scaffold/` 覆盖 `index.html`、`config/outline.md`、`config/share-pages.json`，并写入根目录 `README.md`（约 7 页示例结构） |
| 4. 清理残留 | 删除课例导航 JSON、封面大图、部署密钥、参考文稿、测试/debug 图片、`contents/` 内 PDF 等 |
| 5. 可选 Git | 完成后可选择在新区执行 `git init` |

**不会修改**现有的 `Project/` 目录。

---

## 最小示例页维护位置

```text
Project/tools/dev/new-deck-scaffold/
├── index.html
├── DECK-README.md          → 复制为新 deck 根目录 README.md
└── config/
    ├── outline.md
    └── share-pages.json
```

改示例结构时编辑上述文件即可；下次新建 deck 会自动带上。

---

## 行为与限制

| 项 | 说明 |
|----|------|
| 目标位置 | 工坊根目录下的新文件夹（`Project` 的上一级） |
| 已存在目录 | **不覆盖**；同名则报错退出 |
| 保留名 | 不可使用 `Project`、`template`、`starter`、`draft`、`secrets`，或含 `..` 的名称 |
| 非法字符 | 文件夹名不能包含 `\ / : * ? " < > \|` |
| 复制失败 | `robocopy` 退出码 ≥ 8 时尝试删除半成品目录 |
| 完成后 | 可选 `git init`；默认用资源管理器打开新目录（输入 `n` 跳过） |

## 创建之后

1. 阅读新目录 `README.md`（来自脚手架 `DECK-README.md`）
2. 编辑新目录中的 [`config/outline.md`](../config/outline.md)（写法见 [`docs/guides/OUTLINE_GUIDE.md`](../docs/guides/OUTLINE_GUIDE.md)）
3. 将 PDF / 图片 / 文稿放入 `contents/`
4. 按 outline 修改 `index.html`（标题、议程、`section id`）
5. 登记可分享页：[`config/share-pages.json`](../config/share-pages.json)
6. 预览：进入新目录，双击 `start-lan-server.bat`（**勿**用 `file://` 打开）
7. 远程翻页（可选）：`node remoteNavigator/scripts/generate-deck-nav.js` → `remoteNavigator/start-remote-server.bat`

PDF 抽图与压缩：

```bat
python tools/python/extract_pdf_images.py --pdf contents/你的.pdf --out assets/pdf-extracted
python tools/python/compress_web_assets.py
```

---

## 可选：在新目录初始化 Git

运行脚本时选择 `y` 即可；或事后手动：

```bat
cd 你的演示文件夹名
git init
```

新 deck 默认**不带** `.git`（参考 `Project/` 的 `.git` 不会被复制）。

## 故障排查

| 现象 | 处理 |
|------|------|
| 新目录出现在 `Project/` 里面 | 请更新到最新 `new-deck-from-template.bat`（旧版路径拼接 bug 已修复） |
| 复制后仍有大量 `pdf-extracted` | 同上；新版用目录名 `pdf-extracted` 排除 |
| `python` 未找到无法预览 | 安装 Python 3 并勾选 Add to PATH，或使用仅本机 `start-local-server.bat` |
